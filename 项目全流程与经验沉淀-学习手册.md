# 编程规划智能考核系统 — 项目全流程与经验沉淀（学习手册）

> 本文面向：**接手维护的同事、复用架构的开发者、或培训业务方技术对接人**。  
> 路径以仓库根目录 `programming-planning-assessment/` 为准。

---

## 一、项目是做什么的

- **业务**：编程教培场景下的「规划话术」考核——老师信息、学员画像、**逐字稿（≥800 字）**、两道**模拟家长追问**书面作答，系统按规则引擎给出**学习规划 / 赛考规划 / 答疑能力**三项分数与文字反馈。
- **角色**：学员（或受训老师）完成表单与作答；**培训师**用管理员入口查看历史、筛选、导出 JSON/CSV。
- **数据**：可仅存浏览器 **localStorage**；也可经 **Node 服务**写入 **Turso（libSQL）** 云端数据库，便于多人、异地汇总。

---

## 二、技术架构一览

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 原生 HTML/CSS、ES Module | `index.html` + `js/app.js` 等，无构建步骤 |
| 规则引擎 | `js/scoring.js` | `scoreAssessment()`：三项子分 + 权重合成总分 + 分项亮点/风险 |
| 题库 | `js/questions.js` | 从文档池按逐字稿关键词匹配排序后随机抽 2 题；展示为 `docAsk` 纯问句 |
| 本地存储 | `js/storage.js` | `loadRecords` / `saveRecord`：根据运行环境走 localStorage 或 `fetch('/api/...')` |
| 服务端 | `server/server.mjs` | Express：静态托管上级目录 + `GET/POST/DELETE /api/records` |
| 数据库 | Turso | `@libsql/client`；表 `assessment_records`：`payload` JSON + 冗余列便于 SQL 统计 |

**密钥原则**：Turso Token、管理员口令等**只出现在服务端环境变量**（`server/.env`），**绝不**写进前端仓库或发给学员。

## 三、学员/受训者使用流程（产品路径）

1. **步骤 1 · 信息录入**：考核老师姓名/城市；学员姓名、年龄、性别、年级、省市等。  
2. **步骤 2 · 逐字稿**：粘贴话术，字数不足无法进入下一步。  
3. **步骤 3 · 家长答疑模拟**：系统展示 2 道追问（仅题干）；若服务端开启了 `WRITE_TOKEN`，在**本页「考场保存口令」**填好后点「生成智能评分」。  
4. **步骤 4 · 结果**：展示总分、分项条形图、综合评语与分项反馈。  
5. **自动存档**：点击「生成智能评分」后，前端在出分**之后立即**调用 `saveRecord`；成功则结果页下方提示已保存至本机或服务器；失败会 Toast 说明原因。

**交卷含义**：以「生成智能评分且自动保存成功」为准，不再单独点保存按钮。

## 五、环境如何判断「写本机还是写库」

逻辑在 `js/storage.js` 的 `apiPrefix()`：

1. 若显式设置 `window.__PPAIS_API__`（`null`/`false` 强制本机；`""` 表示同域相对路径；字符串为跨域 API 根）。  
2. 若主机名为 `*.trycloudflare.com`（Cloudflare Quick Tunnel），走同域 `/api`。  
3. 若端口为 `3847`（本机默认 `npm start`），走同域 `/api`。  
4. 否则走 **localStorage**。

**经验**：新增部署域名（如 `*.onrender.com`）时，要么在 `apiPrefix()` 增加识别，要么在 `index.html` 里为学员页面统一设置 `window.__PPAIS_API__ = ""`，否则会出现「以为已上云实际只写了 localStorage」的现象。

## 六、服务端与数据库要点

- **启动顺序**：`server.mjs` 中必须先 `const app = express()`，再注册 `app.use` / 路由；曾出现迁移代码插入后漏写 `app` 导致整站起不来的情况。  
- **表结构**：`id, payload, created_at` + 可选列 `teacher_name, teacher_city,score_total`；启动时用 `PRAGMA` + `ALTER` 做兼容迁移，并用 `json_extract(payload, ...)` 回填旧行。  
- **POST 写入**：从 body 解析记录，同时写入冗余列便于 Turso 控制台直接 `SELECT`。  
- **安全**：`ADMIN_TOKEN` 校验管理读删；可选 `WRITE_TOKEN` 校验写入；CORS 默认较宽，生产可按 `CORS_ORIGIN` 收紧。

## 七、部署与运维路径（给实施人员）

| 场景 | 做法摘要 |
|------|----------|
| 本机演示 | `server` 下 `npm start`，浏览器访问 `http://127.0.0.1:3847/` |
| 纯静态 + 无云库 | `启动本地服务.bat`（Python 8765），数据只在各浏览器 localStorage；若需 Turso 须在页面配置 `__PPAIS_API__` 指向 Node |
| 公网 + 免绑国际卡 | **Cloudflare Quick Tunnel**：本机 `npm start` + `cloudflared tunnel --url http://127.0.0.1:3847`，把生成的 `https://*.trycloudflare.com` 发给学员；**电脑与隧道进程需常开** |
| 公网 + 托管平台 | GitHub + Render 等：Root Directory 填 `server`，环境变量与本地 `.env` 对齐；注意平台是否要求绑卡 |

**Git 与密钥**：`.env` 已在 `server/.gitignore`；推送用 PAT；PowerShell 5 下含中文的 `.ps1` 易出现编码解析错误，故提供英文版 `push-with-token.ps1`。

## 八、产品迭代中的决策记录（可复用的经验）

1. **自动保存**：减少「忘记点保存」导致无记录；口令框前移到步骤 3，与「生成评分即入库」一致。  
2. **题干展示**：去掉逐字稿挂钩长文案，仅展示 `docAsk`，降低阅读噪音、便于考试计时。  
3. **存档结构**：除扁平字段外增加 `scoreResult`、`questionItems`、`scriptMeta` 等，便于事后审计与对账。  
4. **多端一致**：管理员列表与导出依赖 `loadRecords` 异步；远程模式下需先登录再带 `X-Admin-Token`。  
5. **页脚文案**：去掉「仅 localStorage / 启动 bat」等易过时的说明，避免与 Turso 模式矛盾。

## 九、常见问题（FAQ）

**Q：保存成功但 Turso 里没有？**  
A：检查 `apiPrefix()` 是否判定为远程；隧道域名是否已纳入判断；浏览器是否走了错误端口或 `file://`。

**Q：POST 401？**  
A：服务端设置了 `WRITE_TOKEN` 但步骤 3 未填或错误。

**Q：管理员列表为空？**  
A：`ADMIN_PASSWORD` 与 `ADMIN_TOKEN` 不一致；或未登录管理员导致未带 Token。

**Q：Render 绑不了境外卡？**  
A：改用 Cloudflare Tunnel 或国内轻量云 + 人民币支付等替代路径（见仓库内 txt）。
